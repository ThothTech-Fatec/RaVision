Atue como um Desenvolvedor Full-Stack Sênior (Java/Spring Boot e Vue.js/TypeScript). Sua tarefa é desenvolver a Etapa 01 de um sistema de "Gerenciamento de Regras de Negócio e Comissionamento", focada exclusivamente na ingestão e estruturação inicial dos dados.
1. Contexto da Tarefa (User Story Rank 1)
"Como Administrador de Regras, quero uma interface web para importar as planilhas mensais de RH, Vendas e Comissões, para que o sistema tenha os dados exatos para calcular o mês de competência".

2. Stack Tecnológico Obrigatório
Backend: Java 17+ com Spring Boot.

Banco de Dados: PostgreSQL (usando Spring Data JPA / Hibernate).

Frontend: Vue.js (SPA) com TypeScript.

Boas Práticas: Testes unitários (JUnit/Jest) com no mínimo 70% de cobertura de código. O código deve estar limpo e livre de fragmentos comentados.

3. Modelagem de Dados (PostgreSQL / JPA Entities)
Você deve criar as entidades JPA para armazenar os dados exatos de três planilhas. Mapeie os seguintes campos de acordo com a documentação oficial:

A) Entidade BaseRH
Date_Ref (Data da competência)
Cod_Marca (Inteiro)
Descr_Marca (String)
Cod_Loja (Inteiro)
Descr_Loja (String)
Matricula (String - Ex: "MATRIC-1")
Data_Admiss (Data)
Data_Demiss (Data - Pode ser nulo)
Cod_Cargo (Inteiro - Ex: 100, 150)
Descr_Cargo (String - Ex: "VENDEDOR LOJA")
B) Entidade BaseVendas
Date_Ref (Data da competência)
Cod_Marca (Inteiro)
Descr_Marca (String)
Cod_Loja (Inteiro)
Descr_Loja (String)
Matricula (String)
Vlr_Venda (BigDecimal/Double)
C) Entidade BaseComissionamento
Cod_Marca (Inteiro)
Descr_Marca (String)
Cod_Cargo (Inteiro)
Descri_Cargo (String)
%_Comiss (BigDecimal/Double - Ex: 2,50%)

4. Requisitos do Backend (Spring Boot)
Criação de Endpoints REST: Crie os controllers para receber o upload de arquivos (MultipartFile). Podem ser endpoints separados (ex: /api/upload/rh) ou um único endpoint que receba as três planilhas de uma vez para o mês de competência.
Processamento: Crie os serviços (Services) utilizando bibliotecas como Apache POI (para .xlsx) ou OpenCSV (para .csv) para ler os arquivos e salvar os registros em lote no PostgreSQL.
Regra de Negócio Crítica: O processamento deve garantir que os dados importados respeitem a data de competência de cada base (ex: Base RH mês 1 só pode cruzar com Base de Vendas mês 1).

5. Requisitos do Frontend (Vue.js + TypeScript)
Interface de Upload: Crie um componente limpo e responsivo para o "Administrador de Regras"

fazer o upload dos 3 arquivos.
Validação Front-end: Verifique se os arquivos inseridos possuem as extensões corretas antes de enviar para a API.
Feedback Visual: Adicione estados de carregamento (loading spinners) enquanto o backend processa os arquivos e exiba alertas de Sucesso ou Erro ao final.

6. Instruções de Saída para a IA
Por favor, gere os códigos na seguinte ordem estruturada:
Modelagem JPA (As 3 Entidades e seus Repositories).
Backend Services (A lógica de leitura do arquivo e salvamento no banco).
Backend Controllers (Os endpoints de API).
Frontend (Componente Vue.js para a tela de upload e o serviço Axios/Fetch de comunicação).
Testes unitários básicos para o serviço de upload (garantindo o caminho para os 70% de cobertura)
