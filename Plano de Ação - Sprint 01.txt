Plano de Ação - Sprint 01
Etapa 1: Interface de Importação e Banco de Dados (Atende ao Item 1 do Backlog)
Ação: Construir a interface web para o Administrador de Regras fazer o upload das planilhas mensais de RH, Vendas e Comissões.
Tecnologias:
Vue.js (TypeScript): Cria a tela de upload
. (Check DoR: Garantir que os wireframes/mockups desta tela estejam aprovados
).
Spring Boot (Java): Cria a API que recebe esses arquivos e os lê.
PostgreSQL: Recebe os dados brutos e estruturados dessas 3 planilhas. (Check DoR: O modelo de dados das tabelas base deve estar definido
).

Etapa 2: Motor de Cálculo das Comissões Básicas (Atende ao Item 3 do Backlog)
Ação: Desenvolver o código que cruza as bases (pela matrícula para vendedores e código da loja para gerentes) e processa a remuneração com base nas vendas e nas informações padrão de comissionamento.
Tecnologias:
Spring Boot (Java): Aqui criaremos a lógica determinística. O sistema vai aplicar os percentuais de comissão base de acordo com os cargos (ex: venda da loja inteira para o cargo 150 - Gerente) e salvar o valor inicial gerado no banco de dados
.
PostgreSQL: Armazena os resultados calculados desta etapa.

Etapa 3: Motor de Cálculos Proporcionais e Intercorrências (Atende ao Item 4 do Backlog)
Ação: Expandir o motor em Java para interceptar o cálculo base e aplicar as regras proporcionais da legislação trabalhista.
Tecnologias:
Spring Boot (Java): O sistema verificará nas datas de RH se houve férias, admissão ou demissão no mês
. O algoritmo fará a conta proporcional: pega o valor total do comissionamento, divide pelos dias do mês, e multiplica pelos dias efetivamente trabalhados
. O Java gravará esse novo resultado final no banco.

Etapa 4: Assistente e Interface de Chat (Atende ao Item 2 do Backlog)
Ação: Criar a interface por onde o Analista de Operações vai conversar com a IA para perguntar diretamente sobre os valores de comissionamento de um funcionário
.
Tecnologias:
Vue.js (TypeScript): Construção visual do chat
.
Python com LangChain/LlamaIndex: Vai ser o elo de conexão
. Ele vai conectar a pergunta feita no chat com os resultados exatos que o Java salvou no PostgreSQL nas Etapas 2 e 3.
API Hugging Face (LLM): Vai ler os dados trazidos pelo LangChain e formular a resposta em linguagem natural (Ex: "O comissionamento do funcionário MATRIC-58 é de X reais neste mês"), para que o usuário sinta que "o assistente" calculou e devolveu os valores.

Etapa 5: Validação e Qualidade (Checklist de Entrega)
Ação: Garantir que tudo que foi construído atende à qualidade exigida pela Dom Rock.
Tecnologias/Práticas:
GitHub Flow: Todo o desenvolvimento dessas etapas deve ser feito em branches separadas (ex: feature/importacao-arquivos, feature/calculo-base) e ser mesclado (merge) via Pull Request para a versão estável (main)
.
Testes (JUnit/Jest): Desenvolver testes de unidade para todo o backend e frontend com cobertura mínima de 70%
.
